# 5.1 五分钟体验 PromptX

> 先感受，再理解

学习任何新工具的最好方式是：先用起来，感受它的魅力，然后再去理解它的原理。

本节不讲理论，只做三件事：
1. 安装 PromptX
2. 启动服务
3. 体验核心功能

---

## 第一步：安装 PromptX

### 方式一：npx 直接运行（推荐）

无需安装，直接运行：

```bash
npx -y @promptx/cli
```

### 方式二：全局安装

```bash
npm install -g @promptx/cli
promptx
```

### 方式三：配置 Claude Code / Cursor

在 Claude Desktop 配置文件中添加 MCP 服务：

**macOS**: `~/Library/Application Support/Claude/claude_desktop_config.json`
**Windows**: `%APPDATA%\Claude\claude_desktop_config.json`

```json
{
  "mcpServers": {
    "promptx": {
      "command": "npx",
      "args": ["-y", "@promptx/mcp-server"]
    }
  }
}
```

配置后重启 Claude Desktop，即可在对话中使用 PromptX 功能。

---

## 第二步：发现可用资源

启动后，第一个命令是**发现**（discover）——看看有什么可用的角色和工具。

### 使用 discover 工具

```
你：帮我看看有什么可用的角色
```

PromptX 会返回所有可用资源的清单：

```
📦 系统角色（来自 PromptX 核心）
├── luban    - 鲁班：AI 工具集成专家
├── nuwa     - 女娲：AI 角色创造专家
├── sean     - Sean：deepractice.ai 创始人，矛盾驱动决策
└── writer   - Writer：专业文案写手

🔧 可用工具
├── tool://filesystem  - 文件操作
├── tool://pdf-reader  - PDF 阅读
├── tool://excel-tool  - Excel 处理
└── tool://word-tool   - Word 文档处理
```

## 第三步：激活一个角色

让我们激活 **Writer** 角色，体验"去 AI 味"的写作。

### 激活角色

```
你：激活 writer 角色
```

PromptX 会加载 Writer 的完整配置：

```
✅ 已激活角色：Writer

📊 记忆网络图
┌─────────────────────────────────────┐
│  写作 ─── 真实 ─── 细节            │
│    │       │       │               │
│  ISSUE ─ 故事 ─── 场景             │
│    │       │       │               │
│  去AI味 ─ 个人化 ─ 表达            │
└─────────────────────────────────────┘

📚 专业知识：ISSUE 写作范式、真实细节技巧、去 AI 味方法
⚖️ 行为准则：使用真实细节、避免空洞形容词、保持个人风格
🧠 认知策略：recall → 思考 → remember
```

### 体验 Writer 的写作风格

现在，让 Writer 帮你写一段文案：

```
你：帮我写一段关于程序员加班的朋友圈文案
```

**普通 AI 的回答**（AI 味十足）：

> 在这个快节奏的时代，作为一名程序员，我深知努力的重要性。今夜，又是一个奋斗的夜晚。虽然疲惫，但看到代码一行行完成，内心充满了成就感。加油，追梦人！

**Writer 的回答**（真实、有细节）：

> 凌晨两点，第三杯美式见底。旁边工位的老王已经睡着了，键盘上还留着他最后一行注释：// TODO: 明天再改
>
> bug 还是那个 bug，咖啡还是那个咖啡。不同的是，今天是周五。

看到区别了吗？Writer 不会堆砌"努力""奋斗"这样的空洞词汇，而是用**具体的场景和细节**来表达。

## 第四步：体验记忆功能

PromptX 的核心能力是**记忆**。让我们体验 recall 和 remember。

### 查看记忆网络

```
你：recall 一下，看看你记得什么关于写作的内容
```

PromptX 会从记忆网络中召回相关内容：

```
🔍 正在召回与"写作"相关的记忆...

找到 3 条相关记忆：

1. [PATTERN] ISSUE 写作范式
   Issue → Story → Solution → Unique → Effect
   强度：0.9 | 最近访问：2天前

2. [ATOMIC] 去 AI 味的核心方法
   用具体数字替代"很多"，用场景描写替代形容词
   强度：0.8 | 最近访问：1周前

3. [LINK] 真实细节与读者信任的关系
   真实的细节 → 可信度提升 → 读者共鸣
   强度：0.7 | 最近访问：2周前
```

### 保存新记忆

当你告诉 Writer 一些重要信息时，它会自动记住：

```
你：我的目标读者是 95 后程序员，喜欢自嘲和段子

Writer：明白了，我记下来了。以后给你写文案时会考虑这些偏好。
```

PromptX 会自动执行 remember：

```
💾 已保存新记忆

内容：用户的目标读者是 95 后程序员，偏好自嘲和段子风格
概念：用户偏好 目标读者 95后 程序员 自嘲 段子
类型：LINK（关系连接）
强度：0.8
```

下次对话时，Writer 会自动 recall 这个偏好，调整写作风格。

## 第五步：体验角色创建（Nuwa）

如果内置角色不满足需求，可以用 **Nuwa** 创建自定义角色。

### 激活 Nuwa

```
你：激活 nuwa 角色
```

### 描述你的需求

```
你：我需要一个 AI 助手，帮我审查代码时特别关注安全问题
```

Nuwa 不会直接给你一个角色，而是通过**对话**来理解你的真实需求：

```
Nuwa：好的，让我理解一下你的需求。

1. 你说"审查代码"，是指 Code Review 还是安全扫描？
2. "安全问题"具体指哪些？SQL 注入、XSS、还是更广泛的安全实践？
3. 你们主要用什么语言和框架？
4. 这个角色是给你个人用，还是团队共用？

这些问题会帮助我为你设计一个真正有用的角色。
```

通过几轮对话后，Nuwa 会为你生成一个专属角色配置。

---

## 第六步：体验工具创建（Luban）

需要让 AI 能操作外部系统？用 **Luban** 快速集成工具。

### 激活 Luban

```
你：激活 luban 角色
```

### 描述你的需求

```
你：我想让 AI 能读取公司内部的 Confluence 文档
```

Luban 会引导你完成工具配置：

```
Luban：好的，让我帮你集成 Confluence。

我需要以下信息：
1. Confluence 的 base URL（如 https://your-company.atlassian.net）
2. 认证方式（API Token 还是 OAuth）
3. 你最常访问哪些空间？

别担心安全问题——我会把认证信息保存在本地环境变量中，不会暴露给 AI 模型。
```

---

## 快速回顾

五分钟内，你体验了 PromptX 的核心功能：

| 功能 | 命令 | 作用 |
|-----|------|------|
| **发现** | discover | 查看所有可用角色和工具 |
| **激活** | action(角色ID) | 加载角色的知识、原则、记忆 |
| **召回** | recall(关键词) | 从记忆网络检索相关经验 |
| **记忆** | remember(内容) | 将新经验存入记忆网络 |
