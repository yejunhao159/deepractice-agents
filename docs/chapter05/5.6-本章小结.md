## 本章核心内容回顾

### 5.1 五分钟体验 PromptX

快速上手 PromptX 的核心功能：

| 功能 | 命令 | 作用 |
|-----|------|------|
| 发现 | discover | 查看所有可用角色和工具 |
| 激活 | action(角色ID) | 加载角色的知识、原则、记忆 |
| 召回 | recall(关键词) | 从记忆网络检索相关经验 |
| 记忆 | remember(内容) | 将新经验存入记忆网络 |

### 5.2 语义鸿沟

**核心概念**：语义鸿沟是 AI 不知道但角色必须知道的信息。

**四个识别测试**：

| 测试 | 问题 | 不是鸿沟 | 是鸿沟 |
|-----|------|---------|-------|
| Google 测试 | 能搜到吗？ | React 组件概念 | 公司组件规范 |
| ChatGPT 测试 | AI 已知吗？ | Python 语法 | 团队代码风格 |
| 上下文测试 | 离开场景有意义吗？ | HTTP 协议 | 项目 API 约定 |
| 特异性测试 | 通用还是特有？ | 设计模式 | 内部架构决策 |

**Chat is All You Need**：
- 人类是决策者，AI 是执行者
- AI 不是软件的一部分，而是软件的使用者
- 语义鸿沟通过"培养"来填补

### 5.3 Nuwa 角色创建

**设计哲学**：
- 问题比身份重要
- 目的比工具重要
- 过程比结果重要

**ISSUE 范式**：
| 阶段 | 含义 |
|-----|------|
| Identify | 识别核心问题 |
| Scope | 界定工作范围 |
| Stakeholders | 确定使用者 |
| Understand | 深入理解痛点 |
| Explore | 探索解决方向 |

**三层架构**：思维层 / 执行层 / 知识层

**四维度**：身份 / 能力 / 原则 / 个性

### 5.4 Luban 工具创建

**核心理念**：
- AI 是用户，人类是指挥官
- 工具是 AI 的装备，不是人类的软件
- 集成优于开发，连接优于创造

**Token 经济学**：
- 索引优于全文
- 分页加载
- 按需筛选
- 渐进式详情

**工具三层结构**：接口描述 / 执行逻辑 / 使用手册

### 5.5 Engram 记忆网络与 Monogent 认知架构

**Engram 数据结构**：

| 属性 | 作用 |
|-----|------|
| content | 原始记忆内容 |
| schema | 概念索引（空格分隔） |
| strength | 记忆强度（0-1） |
| type | ATOMIC / LINK / PATTERN |

**语义网络**：
- Cue：概念节点
- Network：节点组成的图
- 激活扩散：从种子词向外传播

**Monogent 认知架构**：

| 概念 | 定义 |
|-----|------|
| Experience | 认知的原子单位（单子） |
| Substrate | 处理基质（Computation / Generation） |
| Evolution | 演化类型（微演化 / 宏演化） |
| Pipeline | 七阶段处理管道 |

**七阶段处理管道**：
```
感觉 → 知觉 → 表征 → 激活 → 联想 → 回忆 → 整合
```

**认知循环**：
```
DMN 全景扫描 → 多轮 recall 深挖 → 思考回答 → remember 保存
```

## 常见问题 FAQ

### Q1：PromptX 和普通的 System Prompt 有什么区别？

| 维度 | System Prompt | PromptX |
|-----|--------------|---------|
| 角色定义 | 纯文本描述 | 结构化四维度 |
| 知识注入 | 不支持 | 支持 RAG |
| 长期记忆 | 不支持 | Engram 网络 |
| 跨会话 | 不支持 | 支持 |

**核心区别**：System Prompt 是静态的文本，PromptX 是动态的认知系统。

### Q2：Engram 记忆和向量数据库有什么关系？

Engram 是**抽象概念**，向量数据库是**实现方式之一**。

```
Engram 记忆网络（概念层）
    │
    ├── 短期记忆 → 内存存储
    └── 长期记忆 → 可以用向量数据库实现
```

Engram 提供的是记忆的**语义层**：
- 强度（strength）
- 类型（type）
- 关联关系（schema）
- 衰减机制

向量数据库只是存储手段。

### Q3：什么时候应该使用 PromptX？

**适合场景**：
- 需要专业角色（特定领域专家）
- 需要长期记忆（跨会话持续学习）
- 需要私有知识（填补语义鸿沟）
- 需要工具集成（操作外部系统）

**不需要的场景**：
- 一次性简单问答
- 不需要专业深度的通用对话
- 无需记忆的独立任务

### Q4：如何选择合适的记忆类型（ATOMIC/LINK/PATTERN）？

| 内容特征 | 类型 | 例子 |
|---------|------|------|
| 具体事实、实体信息 | ATOMIC | "用户使用 PostgreSQL" |
| 关系、偏好、因果 | LINK | "用户偏好简洁代码" |
| 流程、方法论、经验 | PATTERN | "先写测试，再实现" |

**判断方法**：问自己"这是一个事实、一个关系、还是一个方法？"

### Q5：recall 没有找到相关记忆怎么办？

1. **检查查询词**：是否使用了记忆网络中存在的词？
2. **尝试 DMN 模式**：`recall(role, null)` 查看全景
3. **扩大范围**：使用更通用的关键词
4. **检查记忆强度**：弱记忆可能已被遗忘

## 结语

本章的核心洞察：

> **AI 的能力不仅取决于模型，更取决于上下文。**

语义鸿沟是 AI 应用的核心挑战，PromptX 通过三个维度来填补：

1. **Nuwa（角色）**：让 AI 有专业身份
2. **Luban（工具）**：让 AI 能操作外部系统
3. **Engram（记忆）**：让 AI 能积累经验

掌握了这些，你就能让 AI 从"通用助手"变成"专业专家"。
